### 3. Local PR-Style HITL Architecture Design

**Task:** Design PR-style Human-in-the-Loop workflow for local development

**Context:**
User asked about doing PR-style development against local (non-GitHub) repositories as part of HITL workflow for local development. This addresses the gap where Phase 1A (CLI prompts) blocks automation and Phase 1B (GitHub Issues) requires network access.

**Problem Addressed:**
- Local development without GitHub/internet access
- Testing TSP Orchestrator without external dependencies
- Need for structured review process with audit trail
- Offline/air-gapped environments
- CI/CD pipelines with review gates

**Architecture Designed:**

**Key Features:**
- ‚úÖ Zero infrastructure (pure git-based solution)
- ‚úÖ Works offline (no network required)
- ‚úÖ PR-like experience (branch ‚Üí review ‚Üí merge)
- ‚úÖ Full audit trail (git notes + merge commits)
- ‚úÖ TSP Orchestrator compatible
- ‚úÖ Fast iteration (no network latency)

**Components Designed:**

1. **BranchManager**
   - Create feature branches for agent output
   - Commit agent changes with structured messages
   - Generate human-readable diffs
   - Manage git notes for review metadata

2. **ReviewPresenter**
   - Rich terminal UI for review display
   - Format diffs with syntax highlighting
   - Show quality gate reports in tables
   - Interactive diff viewing

3. **ApprovalCollector**
   - Prompt for approve/reject decision
   - Collect justification (required)
   - Validate user input
   - Capture reviewer metadata

4. **MergeController**
   - Merge approved branches with `--no-ff`
   - Tag rejected branches for historical record
   - Create annotated tags for approvals
   - Preserve full review history

**Workflow Designed:**
```
1. Agent generates output ‚Üí Quality gate fails
2. Create feature branch (review/task-id-gate-type)
3. Commit agent output to branch
4. Generate diff from main branch
5. Present review in terminal (diff + quality report)
6. Human decision: APPROVE / REJECT / DEFER
7. If APPROVED: Merge with --no-ff + tag
8. If REJECTED: Tag branch + optionally delete
9. Store review metadata in git notes
10. Full audit trail preserved in git history
```

**Integration with TSP Orchestrator:**
- Implements `ApprovalService` interface
- Compatible with existing HITL architecture
- Supports custom approval logic (auto-approve, conditional)
- Scriptable for automated testing
